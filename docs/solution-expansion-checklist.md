# JASAIN 솔루션 확장 체크리스트

## CUJASA 콘텐츠 포맷 확장

- 현재: Threads 텍스트 자동화 중심.
- 다음 포맷: 이미지 카드, 짧은 영상 스크립트/캡션, 링크 없는 정보성 글.
- 계정 설정에는 `content_formats` 형태로 `text`, `image`, `video` 선택값을 둔다.
- v1 구현은 실제 이미지/영상 생성 전, 포맷별 프롬프트와 예약 슬롯만 분리한다.

## 어필리에이트 채널 확장

- 베이직 계정은 최대 3개 채널까지 연결한다.
- 1차 채널:
  - `coupang`: 쿠팡 파트너스 Open API. Access Key, Secret Key, Partner ID, Tracking Code 필요.
  - `toss_share_manual`: 토스 쉐어링크 수동 등록. 공식 공개 API 확인 전까지 링크 템플릿/수동 URL 중심.
  - `custom_affiliate`: 기타 제휴 링크. 채널명, 기본 URL, 추적 파라미터, 금지 카테고리 입력.
- 링크 노출 비율은 채널별 가중치로 저장한다. 예: `coupang:1`, `toss_share_manual:1`, `custom_affiliate:2`.
- 포스팅 생성 시 활성 채널 중 가중 랜덤으로 링크 채널을 선택한다.
- 채널별 API 실패 시 같은 채널을 성공 처리하지 않고, 다른 채널 후보 또는 링크 없는 글로 대체한다.

## SPREAD 자동화 방향

- 광고주/사장: 캠페인 등록, 상품/혜택/마감일/선정 기준 입력.
- 신청자: 신청 폼 제출, 채널/URL/팔로워/카테고리/메모 입력.
- 운영자/자동화: 신청 마감 후 후보 리스트 정렬, CSV 다운로드, 선정 상태 변경.
- 제출물 검수: 제출 URL, 필수 키워드, 금지 표현, 마감일 준수 여부를 체크한다.
- v1은 JASAIN 워크스페이스 drawer 안에서 캠페인 추천, 참여자 선정, 제출물 검수를 완성한다.

## 결제/크레딧 원칙

- DEXOR는 크레딧형으로 운영한다.
- SPREAD는 아직 가격 정책 미확정. v1은 무료 사용량 기반으로 두고, 캠페인 생성/선정/검수 중 어떤 행위를 차감할지 별도 확정한다.
- CUJASA는 기존 월정액/영구구매 기반을 유지한다.
